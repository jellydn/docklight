## Findings considered and rejected

- "Rewrite SSH execution to use the shell `ssh` binary everywhere": rejected because `server/lib/executor.ts` already uses `node-ssh` pooling for Dokku commands, and replacing it would broaden risk without addressing issue #129.
- "Make public IP detection stop using `api.ipify.org` entirely": rejected because public IP detection is still useful for domain/global-domain defaults; the problem is reusing it as the internal container-to-host SSH bridge target.
